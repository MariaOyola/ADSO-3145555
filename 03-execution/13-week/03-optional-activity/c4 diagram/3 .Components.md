# DDD вҖ” Domain Driven Design

Es una arquitectura avanzada usada en sistemas empresariales grandes.

La idea principal es:

> El negocio es el centro del proyecto.

O sea:
la lГіgica del negocio se separa completamente de:

- la base de datos
- los controladores
- Spring Boot
- APIs

---

# ВҝPor quГ© es difГӯcil de entender al inicio?

Porque ya no se organiza como:

- Controller
- Service
- Repository

sino que ahora el proyecto se divide por responsabilidades reales del sistema.

DDD intenta representar el negocio como funciona en la vida real.

---
