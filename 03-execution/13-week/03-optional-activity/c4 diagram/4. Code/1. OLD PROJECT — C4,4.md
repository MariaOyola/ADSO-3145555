# ¿Qué lleva cada componente?

# Entity

Representa las entidades del sistema.

```txt
Entity
│
├── Instructor
│   ├── atributos
│   ├── constructores
│   ├── getters/setters
│   ├── relaciones
│   └── validaciones básicas
│
├── Ambiente
├── Horario
├── Ficha
└── Observacion
```

Aquí normalmente va:

- clases
- atributos
- constructores
- getters/setters
- relaciones entre entidades

---

# Repository

Encargado de acceder a la base de datos.

```txt
Repository
│
├── InstructorRepository
│   ├── save()
│   ├── findById()
│   ├── delete()
│   ├── findAll()
│   └── consultas personalizadas
│
├── HorarioRepository
└── AmbienteRepository
```

Aquí normalmente va:

- interfaces
- métodos CRUD
- consultas personalizadas

# Service


- clases service

- comunicación con repositories

# DTO

Objetos para transferencia de datos.

```txt
DTO
│
├── Request
│   ├── CrearHorarioRequest
│   └── CrearInstructorRequest
│
└── Response
    ├── HorarioResponse

```

Request normalmente lleva:

- atributos enviados por el cliente
- validaciones

Response normalmente lleva:

- datos que devuelve el sistema
- estructuras simplificadas
